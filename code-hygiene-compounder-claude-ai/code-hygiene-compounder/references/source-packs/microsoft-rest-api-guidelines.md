# Source Pack: Microsoft REST API Guidelines

- Source ID: `microsoft-rest-api-guidelines`
- Activation: REST API
- Official link: https://github.com/microsoft/api-guidelines

## Distilled Checks

- For REST API changes, check consistency, compatibility, predictable naming, versioning, pagination, filtering, error shape, and documentation.
- Avoid breaking clients unless versioning/migration is explicit.
- Use consistent HTTP methods, status codes, and resource modeling.
- Make error responses actionable and stable.
- Update docs/examples when request or response behavior changes.
