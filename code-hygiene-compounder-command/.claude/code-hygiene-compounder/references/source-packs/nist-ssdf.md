# Source Pack: NIST SSDF

- Source ID: `nist-ssdf`
- Activation: always
- Official link: https://csrc.nist.gov/pubs/sp/800/218/final

## Distilled Checks

- Consider secure development as lifecycle work: prepare, protect software, produce well-secured software, and respond to vulnerabilities.
- Verify the change reduces vulnerability risk rather than merely hiding symptoms.
- Prefer repeatable practices that prevent recurrence: tests, checks, threat modeling, secure defaults, and documented response.
- Protect source, build, dependencies, and release artifacts when relevant.
- Record security assumptions and residual risk.
- Include vulnerability response implications for public or deployed software.
