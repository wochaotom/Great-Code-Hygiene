# Source Pack: OpenTelemetry

- Source ID: `opentelemetry`
- Activation: observability
- Official link: https://opentelemetry.io/docs/concepts/

## Distilled Checks

- When observability is relevant, consider logs, metrics, traces, context propagation, and correlation.
- Instrument behavior that operators need to diagnose failures, latency, saturation, retries, and user impact.
- Avoid sensitive data in telemetry.
- Prefer existing instrumentation libraries and semantic conventions used by the repo.
- Ensure added telemetry has stable names and useful attributes.
