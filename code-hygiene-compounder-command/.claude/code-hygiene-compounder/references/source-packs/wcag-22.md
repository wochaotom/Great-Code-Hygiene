## Distilled Checks

- For web UI changes, check perceivable, operable, understandable, and robust behavior.
- Verify labels, names, roles, focus order, keyboard operation, target size, contrast, visible errors, and semantic markup where relevant.
- Prefer WCAG 2.2 resources because they include earlier 2.x success criteria except obsolete parsing details.
- Use automated accessibility tests as a starting point, not the whole review.
- Include manual interaction checks for keyboard/focus/error behavior when risk warrants.
